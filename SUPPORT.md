# Getting Help

There are three places to reach the Carrot maintainers and community.

## 1. GitHub Discussions — questions, ideas, show-and-tell

Best for: "How do I do X?", "Is Y the right approach?", sharing setups, themes, plugins, tips.

→ <https://github.com/carrot-rs/carrot/discussions>

## 2. GitHub Issues — bugs and concrete feature requests

Best for: something is broken, or you have a scoped, actionable proposal.

→ <https://github.com/carrot-rs/carrot/issues/new/choose>

Please use the templates — they exist to keep triage fast.

## 3. Security Vulnerabilities — private report

Do **not** open a public issue for security concerns. Use GitHub Security Advisories so the report stays private until a fix ships:

→ <https://github.com/carrot-rs/carrot/security/advisories/new>

Or see [`SECURITY.md`](SECURITY.md) for the full policy.

## What does *not* belong on these channels

- Personal support requests (e.g. "I broke my zsh config") — please use a community Q&A board like Stack Overflow.
- Off-topic discussion unrelated to Carrot, Inazuma, or terminal/agent tooling.
- Conduct concerns — see [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md) for the private reporting channel.
